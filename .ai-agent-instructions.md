# AI Agent Instructions for Master Template (https://github.com/FlutschiCH/FRONTEND-TEMPLATE-DONT-DELETE-.git)

Welcome! You are an expert frontend AI assistant working on this React + TypeScript + Tailwind CSS project. Please follow these guidelines strictly to ensure consistency and prevent hallucinations.

## 🚀 Tech Stack

- **Framework:** React 18+ (Vite)
- **Language:** TypeScript
- **Styling:** Tailwind CSS
- **UI Components:** `shadcn/ui` (Radix UI primitives + Tailwind)
- **Routing:** `react-router-dom` (v6+)
- **Animations:** `framer-motion`
- **Absolute Imports:** Configured to use `@/` for the `src/` directory.

## 📁 Project Structure

- `@/pages/` - All route-level page components.
- `@/components/layout/` - Structural components like `Navbar`, `Footer`, `Sidebar`.
- `@/components/ui/` - `shadcn/ui` components and other reusable, atomic UI elements.

## 🎨 Design System & Tailwind Guidelines

This project uses a dark-mode, "glassmorphism" aesthetic primarily utilizing the `sky` and `zinc`/`white` color palettes.

**CRITICAL: Always use these exact class combinations to maintain design consistency.**

- **Page Wrapper:** Every page must be wrapped in:
  `className="flex flex-col items-center justify-center min-h-[calc(100vh-16rem)] p-6"`
- **Main Glass Card (Outer):**
  `className="bg-sky-950/20 border border-sky-400/10 backdrop-blur-xl p-10 rounded-[2.5rem] w-full max-w-4xl mx-auto text-center space-y-6 tracking-tight font-medium"`
- **Inner Glass Cards (Widgets/Grids):**
  `className="bg-sky-900/30 border border-sky-400/20 rounded-[1.5rem] p-6 text-sky-400/70"`
- **Typography:**
  - **H1:** `text-4xl font-bold text-white mb-4`
  - **Subtitles/Descriptions:** `text-lg text-white/80 leading-relaxed`
  - **Body text:** Use `text-white/70` for standard text and `text-sky-400` for highlights/accents.

## 🧩 Component Rules

1. **Named Exports Only:** All components must use named exports (`export const MyComponent = () => {}`). Do NOT use `export default` (except for `App.tsx` and `main.tsx`).
2. **UI Components (shadcn/ui):** Heavily rely on `shadcn/ui` components. Do not build raw HTML interactive elements (like `<button>`, `<input>`, `<dialog>`, etc.). Always import them from `@/components/ui/*`. If a required shadcn component seems to be missing, kindly ask the user to install it via `npx shadcn-ui@latest add <component-name>`.
3. **Routing:** Always use `<NavLink to="...">` from `react-router-dom` for internal navigation, never raw `<a>` tags.
4. **Forms & Custom Styling:** When overriding or extending `shadcn/ui` components, maintain the glassmorphism style where appropriate: `bg-sky-900/30 border-sky-400/20 text-white`.

## 🏗️ Workflow for Adding a New Page

When asked to create a new page, follow these steps:

1. Create the new file in `src/pages/NewPage.tsx`.
2. Export it as a named component.
3. Implement the UI using the strict glassmorphism classes mentioned above.
4. Update `src/App.tsx` to import the new page and add its `<Route>` inside the `<Routes>` block.
